post /register
Registra un nuevo usuario. (registerUser)
Registra un usuario con un correo electrónico y una contraseña. Si el correo ya existe, devuelve un fallo.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body register_body (required)
Body Parameter — Información del usuario a registrar.

Return type

String

Example data

Content-Type: application/json
"REGISTER_SUCCESS"

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Registro exitoso. String

400

Fallo al registrar (correo ya existe). String
Up 
post /validateEmail
Valida si un correo electrónico está registrado. (validateEmail)
Valida si el correo electrónico proporcionado está registrado en el sistema.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body validateEmail_body (required)
Body Parameter — Correo electrónico a validar.

Return type

String

Example data

Content-Type: application/json
"EMAIL_VALID"

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Correo electrónico válido. String

404

Correo electrónico no encontrado. String
Up 
post /validatePassword
Valida la contraseña para un correo electrónico registrado. (validatePassword)
Valida si la contraseña proporcionada es correcta para el correo electrónico registrado.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body validatePassword_body (required)
Body Parameter — Credenciales del usuario a validar.

Return type

String

Example data

Content-Type: application/json
"PASSWORD_VALID"

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Contraseña válida. String

401

Contraseña incorrecta. String
